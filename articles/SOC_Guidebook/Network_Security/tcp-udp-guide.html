TCP vs UDP: What Analysts Detect in Network Traffic

A practical SOC analyst guide to recognizing TCP and UDP attack patterns in packet captures, logs, and SIEM alerts.

Understanding the differences and security implications of Transmission Control Protocol (TCP) and User Datagram Protocol (UDP) is foundational for defensive cybersecurity roles. These protocols carry most internet traffic, from legitimate web requests to attacker command-and-control (C2) beacons. Analysts must recognize normal patterns to detect protocol exploitation during triage, hunting, and incident response.

TCP and UDP appear constantly in firewall logs, packet captures, and Security Information and Event Management (SIEM) alerts. Attackers weaponize both protocols in multiple ways. This article covers the fundamentals of TCP and UDP, common attack examples, and practical analyst workflows. By the end of this guide, you'll have a clearer idea of what to look for in TCP and UDP traffic and where it appears in logs and packet captures, so you can start developing an analyst's eye for abnormal behavior.

What They Are

TCP establishes reliable connections through a three-way handshake where devices exchange SYN, SYN-ACK, and ACK packets before data transmission begins. TCP then uses sequence numbers to detect lost packets, trigger retransmissions, and ensure ordered delivery. Analysts rely on this predictable connection lifecycle (often described as a TCP state machine) when baselining normal web traffic patterns. These predictable patterns help analysts baseline normal web traffic.

UDP skips all connection overhead for maximum speed. There's no handshake—just datagrams with source/destination ports and a basic checksum. Lost packets disappear silently with no retransmission or guaranteed ordering. This makes UDP perfect for DNS queries and streaming, but a challenge for analysts hunting volume-based attacks.
